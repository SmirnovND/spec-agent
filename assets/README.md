# Assets — ресурсы для spec-agent

## Структура

### `examples/` — примеры спецификаций
Готовые примеры спецификаций для разных типов компонентов:
- **controller_spec.md** — HTTP контроллер/обработчик запросов
- **usecase_spec.md** — бизнес-логика и сценарии использования
- **service_spec.md** — бизнес-сервисы и операции
- **repository_spec.md** — доступ к данным

Используйте эти примеры как:
- Шаблоны для создания новых спецификаций
- Справку по правильному оформлению (структура разделов, ссылки)
- Эталон качества спецификаций

### `prompts/` — промты и правила для работы с агентом (встроены в `.spec_agent/`)
Ключевые документы для понимания spec-driven архитектуры встраиваются при инициализации проекта:
- **spec_rules.md** — правила создания и оформления спецификаций
- **agent_prompt.md** — инструкции для AI-агента, работающего со спецификациями
- **workflow.md** — жизненный цикл работы агента при изменении спецификаций

После `spec-agent init` файлы будут в `.spec_agent/prompts/`

## Как использовать

### Настройка AI-агента
1. Выполните `spec-agent init`
2. Прочитайте `.spec_agent/prompts/agent_prompt.md`
3. Используйте `.spec_agent/prompts/spec_rules.md` как справочник для валидации
4. Следуйте `.spec_agent/prompts/workflow.md` при работе с изменениями

### Работа со спецификациями
1. Изучите `.spec_agent/prompts/spec_rules.md` для понимания правил
2. Смотрите примеры в `.spec_agent/examples/`
3. Следуйте структуре и семантике при создании новых спецификаций

## Добавление новых ресурсов

Вы можете добавлять:
- Новые примеры для других типов компонентов (middleware, validator и т.д.)
- Промты для специфических задач (рефакторинг, оптимизация)
- Гайды и лучшие практики
