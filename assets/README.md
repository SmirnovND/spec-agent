# Assets — ресурсы для spec-agent

## Структура

### `examples/` — примеры спецификаций
Готовые примеры спецификаций для разных типов компонентов:
- **controller_spec.md** — HTTP контроллер/обработчик запросов
- **usecase_spec.md** — бизнес-логика и сценарии использования
- **service_spec.md** — бизнес-сервисы и операции
- **repository_spec.md** — доступ к данным

Используйте эти примеры как:
- Шаблоны для создания новых спецификаций
- Справку по правильному оформлению (структура разделов, ссылки)
- Эталон качества спецификаций

### `prompts/` — промты для LLM
Готовые промты для работы с LLM (GPT, Claude и т.д.):
- **create_spec.md** — как написать новую спецификацию
- **analyze_spec.md** — как проанализировать существующую спецификацию

Используйте при:
- Генерации спецификаций через AI
- Проверке и улучшении спецификаций
- Обучении новых разработчиков

## Как использовать

### Создание новой спецификации
1. Прочитайте `prompts/create_spec.md`
2. Смотрите примеры в `examples/`
3. Следуйте структуре и правилам

### Проверка спецификации
1. Используйте `prompts/analyze_spec.md` как чек-лист
2. Попросите AI проанализировать вашу спецификацию
3. Улучшайте согласно рекомендациям

## Добавление новых ресурсов

Вы можете добавлять:
- Новые примеры для других типов компонентов (middleware, validator и т.д.)
- Промты для специфических задач (рефакторинг, оптимизация)
- Гайды и лучшие практики
